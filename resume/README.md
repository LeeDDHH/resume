### 株式会社 BRAIN MAGIC（2020/04〜2022/08）

左手デバイス Orbital2 とつないで使う「Orbital2 Core」の開発に従事。
開発リソースが限られる中で、既存ユーザーの利便性を高める追加機能と新規ユーザーがプロダクトを使い始めやすくするための機能の内部設計から実装まで自走。
得意とするフロントエンド開発の経験を活かして、フロントエンドの改善をリード。
また、実装する機能に関して企画とデザイナーが意見を言いやすいよう、話の場を設けることで意思決定と開発のサイクルを早くする活動も実施した。

- **プロジェクト規模：**
  - 4 人チームでの開発
- **プロジェクト詳細：**
  - Electron + React でのハードウェア連携のデスクトップアプリケーション開発
  - あらかじめ設定したキーに合わせてアプリケーションを起動させる機能の開発
  - Node.js / Electron のバージョンアップによる既存仕様の修正および周辺技術のバージョンアップ
  - フロント全コードの JavaScript から TypeScript へのリプレイス
  - デスクトップアプリケーション内で使⽤できる機能の実装
    - プロファイルの切り替え
    - プリセット
    - チュートリアル
    - Adobe Lightroom Classic 拡張機能
  - ハードウェアの機能をデスクトップアプリケーション側で制御する機能の開発
  - デスクトップアプリケーションにデータとして読み込むためのキーロガーアプリの開発
  - React でのクリエイターの作用可視化サービス開発
- **その他：**
  - 実装時のデザインをデザイナーと折衷
  - 社内ドキュメントの整備（フロントエンドの仕様、ライブラリ選定の理由）

## 意欲・興味

- 少人数チームで小さく早くプロダクトインし、フィードバックを受けながら改善のサイクルを回していくアジャイルな開発スタイルを好みます
- 得意分野のフロントエンドにコミットし、機会があればバックエンドにもコミットしたいです
- ペアプロを積極的に取り入れ、チームで暗黙知の共有や、技術の継承をしていきたいと思っています
- 開発に関わったサービスがユーザーの利便性につながるときと開発に関わったメンバーの開発体験が向上されたとき、それに没頭して続けられます

## 希望条件

- UI 先行で考えるより、UX 先行やユーザーファーストなプロダクトを作っていく過程が好きです
  - 言われたとおりにただ作るのではなく、機能要件に対しても一緒に考えていけるチームだと良いです
- 実装して終わりにならず、実装後の計測や改善について検討できるチームで働きたいです
- 新しい挑戦（技術的なもの・制度的なもの）の際、チームメンバーと相談して小さい単位で試して取り組める環境が好きです
